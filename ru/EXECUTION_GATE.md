# Execution Gate v1

Короткий операционный протокол для стабильного порядка действий в работе агента.

1. Перед каждым действием задавать вопрос: «Какой минимально необходимый следующий шаг?»
2. Если есть явная ошибка компиляции/диагностика, сначала исправлять её без дополнительных проверок в середине цикла.
3. Делать одну контрольную верификацию в конце цикла (`build`/`test`/`diagnostics`), а не каскад промежуточных проверок.
4. Выбирать проверку по типу задачи и файла (например, C# -> Roslyn-диагностики; docs -> линт по необходимости).
5. После завершения задачи фиксировать короткий postmortem: какой один лишний шаг убрать в следующий раз.

Цель протокола: снижать шум, ускорять цикл и закреплять правильный порядок выполнения.
